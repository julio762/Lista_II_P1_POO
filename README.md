## Objetivos

O objetivo central deste trabalho é aplicar e avaliar, de forma prática e rigorosa, os pilares avançados da **Programação Orientada a Objetos (POO)** e de **Arquitetura de Software Modulada** utilizando a linguagem Python. 

---

## Estrutura de Diretórios do Repositório

Conforme as diretrizes organizacionais, cada questão possui o seu próprio ambiente modularizado independente.

Todos os códigos estão organizados em pastas dentro do repositório para facilitar a leitura, manutenção e entendimento do projeto. Cada sistema possui uma estrutura separada em arquivos e diretórios, seguindo uma organização simples baseada em Programação Orientada a Objetos (POO).

A pasta models contém as classes principais do sistema, onde ficam os atributos, métodos, heranças, abstrações e os comportamentos dos objetos. É nela que estão as entidades utilizadas no projeto, como funcionários, mídias, notificadores e outros objetos criados durante os exercícios.

A pasta services é utilizada para armazenar as regras de funcionamento do sistema e os processamentos principais. Nela ficam funções e classes responsáveis por executar ações, processar informações e controlar parte da lógica do programa.

A pasta ui representa a interface com o usuário. Ela contém os menus, mensagens exibidas no terminal e toda a interação feita através do input() e print().

O arquivo main.py é responsável pelo fluxo principal do programa. É nele que o sistema é iniciado, os menus são executados e os testes das funcionalidades são realizados.
